History

The History view is used to recall and edit all of your recorded Splitwatch events and split data.

Filter Tab Bar

The Filter Tab Bar helps you easily recall a specific Splitwatch event.

All

The All tab control lists every recorded Splitwatch event by date with the most recent first. Select an event to display the event and split data (see History: Split Detail View below).

Date

The Date tab control will list all of the dates on which you recorded Splitwatch events. Most recent appear first. Select a date to display a list of events from that date.

Distance

The Distance tab control lists all of the recorded event distances from shortest to longest. Select a distance to display a list of events of that distance.

Event

The Event tab control lists all of the recorded event names alphabetically. Select an event name to display a list of events with that name.

Athlete

The Athlete tab lists all of the recorded athlete names alphabetically. Select an athlete name to display a list of events by that athlete.

Edit Button

The edit button is used to delete events or groups of events from the History database. To delete an event or group of events, tap the Edit button to put the current list into edit mode. Optionally, left swipe a row to bring up the delete button directly. Tap the Done button to exit edit mode. (Note: for split editing see History: Split Edit View below).

Split Detail View

Selecting an event in the History view brings up the Split Detail view. The Split Detail view displays all of the data recorded for that event.

Athlete/Event Name Editing

Select the Athlete and Event text fields or the Name Selector Button to assign or edit the names for this event. If this event was recorded by a Multiwatch then the text fields will display the names assigned to the Multiwatch.

Name Selector Buttons

These buttons bring up a selection wheel with all the athlete or event names in the database. Select a name and tap the Set button to quickly assign the athlete and event name fields.

Split Edit Button

The Split Edit button brings up the split editing view. See Split Edit View below for more detail.

Event Info

Date: The date and time the event started

Distance: Total distance of the event

Time: Total time for the event

Split Column Header

Indicates the split columns for the event. This column header will change depending on the split unit selected for the event (see Settings: Split Units Selector).

Split Detail

Each interval will be represented as a row with appropriate splits rolled up to the right. The split detail view can be scrolled if necessary to view additional rows.

Split Summary

At the bottom of the Split Detail view are the split summary rows. The Last splits display closing times over the last 5 splits. Min, Max, and Avg splits are calculated.

Min/Max Highlight

Tapping any of the min and max values in the split summary will highlight where in the race those splits occured.

Report Button

The Report button allows you to easily export event data by generating and either emailing or printing a Split Report. Tap the Report button to bring you to the Report generator view (below). Here you can set filters to generate and (if you want) combine multiple split reports for emailing or printing to any AirPrint compatible printer.

Event Selectors: There are four event selectors: Athlete, Event, Date and Distance which act as filters for your report. The report generator initially sets the selectors to the single event you selected in the Split Detail view. However, you can include more events in the report by selecting any or all of the selectors and setting them to the 'All' option which will not exclude any events based on that selector. Once you have set the filters then tap either the Email button or the Print button (the Email view is detailed below).

To/CC/Bcc: Assign the recipients by tapping the To/Cc/Bcc fields. This will display a keyboard and the Add Contact button. The Add Contact button lets you select an email address from your list of contacts. If you have added contacts in the Email Setup view then those addresses will be placed in the To field when the Email Composer view is displayed.

Subject: The subject contains the name of the athlete, the event distance and the final time. The subject line can be edited before sending.

Split Report: The body of the email is the Split Report with all the event info and split data, including closing splits. The body of the Split Report can be edited before sending.

Send/Cancel: Hit the Send button to send the email to the recipients or Cancel to cancel and dismiss the Split Report.

History Button

The History button returns you to your last location in the History view.

Event Edit View

The Event Edit view allows you to fully edit event and split data for any event in your database.

Unit/Interval Selctors: Allow you to convert the selected event to a different base unit and interval (see Settings: Split Units Selector for more info). This is useful if you quickly need to start a watch before setting the correct units and lap distance in Settings.

Mode Switches: Allow you to change the mode of the event (see Settings: Kilo Splits Switch and Settings: Furlong Mode for more info).

Split Rows: Select the split to be edited. This brings up the Split Edit view.

Split Edit View

Selected Split: Shows the selected split. This time will update with the Split Time Picker.

Split Time Picker: Spin the dials to change the split time. The format is hh:mm:ss.xx.

Edit Actions: Here you decide what to do with the split: Change Split, Delete Split, Insert New Split, or Cancel. Changed splits must be greater than the previous split and less than the next split. Inserted splits will automatically be placed in the correct position in the event. Tapping one of the edit buttons will return you to the Event Edit view and the edited split will flash red to indicate the change.

Split Detail Button

The Split Detail button returns to the Split Detail view. Any changes made in the Event Edit or Split Edit views will be reflected in the Split Detail View and will be saved in your event database.
